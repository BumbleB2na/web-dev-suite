Load Image jQuery Plugin

Dynamically inserting 'src' attribute and/or detecting when an image has loaded can be unreliable accross browsers, mainly due to how browsers cache image files.
This plugin allows reliable image 'src' injection, allows a callback event to fire when loading is complete, and provides complete control over loading images.

Check it out:

Method 1:

Html & Inline Css:

    <img data-img-src="images/00-thumb-retro-web-dev-series.png" style="display:none;" />

JavaScript:

   $('img').loadImage().fadeIn();

Method 2:

Html & Inline Css:

    <img style="display:none;" />

JavaScript:

   $('img').loadImage({ src: 'images/00-thumb-retro-web-dev-series.png' }).fadeIn();

Plugin Features:

  • chainable jquery plugin
        $('#img1').loadImage().fadeIn();
  • apply to single or multiple elements of a jquery selection
        $('img').loadImage(); // applies to all img elements
  • safely injects an 'src' to an image or a set of images
  • img elements that already have a 'src' attribute are not affected by the plugin, which means you could have img elements with 'src' attributes that load normally and have img elements without a 'src' attribute that you want to defer loading until later.
  • optionally pass in a callback function that triggers when image(s) have completely loaded
        $('img').loadImage({ callback: 'yourFunction' });

Support, Testing, and Requirements:

  • supports IE7+, Chrome, Firefox, Safari and Opera
    • support is backed by extensive unit tests and manual UI testing
  • plugin requires jQuery JavaScript plugin and Modernizr plugin is reccommended.
    • tested with: jQuery 1.7.1 (if using older versions of jQuery, please do so at your own risk)
    • tested with Modernizr 2.0.6 (if using older versions of Modernizr, please do so at your own risk)
